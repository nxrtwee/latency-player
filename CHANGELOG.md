## Что такое Latency

**Latency** — десктопный медиаплеер (Windows) уровня Spotify, написанный с нуля
на **Electron + Vite + React + TypeScript**. В основе — архитектура подключаемых
источников воспроизведения (`PlaybackProvider`): локальные файлы и SoundCloud
работают через единое ядро, новый источник добавляется реализацией одного
интерфейса. Состояние — на `zustand`, без тяжёлых зависимостей.

Это не форк чужого клиента: собственный UI (тёмная тема, неоновый акцент,
3 колонки, кастомная рамка окна), реактивный визуализатор, караоке-тексты,
персональные миксы, личный кабинет и гибкое оформление.

## [0.3.0] — 2026-06-25 — только для ПК

> **Это обновление только для десктопа (Windows).** Мобильные версии (Android/iOS)
> остаются на **0.1.0** без изменений.

### 🟡 Яндекс Музыка — доступ без VPN в России
- Новый источник `PlaybackProvider` (`yandex`): SoundCloud в РФ заблокирован
  Роскомнадзором, а Яндекс Музыка работает без VPN. Вход **своим аккаунтом**
  (OAuth, токен хранится локально), стрим по **вашей подписке Яндекс Плюс**.
  Без входа — 30-сек превью, после входа — полные треки. Без обхода блокировок и DRM.
- Поиск треков и артистов, страницы артистов, подписанные стрим-URL
  (`download-info` + MD5), проксирование через `media://` (EQ + живой визуализатор).
- Переключатель источника **SoundCloud / Яндекс** в поиске — в т.ч. для «поиска по строчке».

### 🌊 Моя волна
- Персональное бесконечное радио Яндекса: отдельная кнопка (в «Моей музыке» и на
  главной при входе через Яндекс), брендовая страница и **самопополняющаяся** очередь.

### ❤️ Лайки
- **Импорт** лайков из SoundCloud / Яндекс Музыки в глобальные лайки (по кнопке в
  настройках) и **удаление** импортированных по сервису.
- Лайки больше не тормозят на 300+ треках (O(1)-проверка вместо скана массива).

### ✨ Прочее
- Несколько авторов у трека — открывается **любой** (не только первый), в т.ч. в правой панели.
- Настоящие логотипы SoundCloud / Яндекс Музыки на треках, в профиле и настройках.
- Genius-тексты теперь статичные (без фейковой синхро) — синхронизируются вручную.
- Профиль показывает оба аккаунта (SC + Яндекс); комментарии переехали в «Обзор»,
  из них убраны дублирующие кнопки и прогресс-бар.

## [0.2.0] — 2026-06-23

### 🔍 Поиск трека по строчке из песни
- Во вкладке **Поиск** (Explore) переключатель: Треки / По строчке.
- Режим **По строчке**: вводишь фрагмент текста → Genius находит песню → по названию
  и автору ищется трек на **SoundCloud** → показывается с обложкой и найденной строчкой.
- Клик — немедленное воспроизведение (все найденные складываются в очередь).
- Треки, не резолвленные в SoundCloud, автоматически отбрасываются.

### 🎛️ Эквалайзер
- **10-полосный графический EQ** (32 Гц … 16 кГц, BiquadFilter peaking) в общем
  Web Audio графе между источником и анализатором.
- Вертикальные ползунки ±12 дБ, тумблер вкл/выкл, **7 пресетов** (Flat, Bass Boost,
  Treble Boost, Vocal, Electronic, Rock, Loudness).
- Работает и на локальных файлах, и на SoundCloud (progressive-треки проксируются
  через `media://remote`, HLS — через MSE same-origin).
- Открывается кнопкой EQ в правой панели.

### 🧠 Автопилот (Autopilot)
- Тумблер на нижнем прогресс-баре (рядом с караоке). Когда включён и очередь
  заканчивается, плеер автоматически находит похожие треки через SoundCloud
  (`scRelated`) и продолжает играть без остановки. При повороте ручки посреди
  доигранной очереди — сразу подхватывает продолжение.

### 💬 Комментарии SoundCloud
- Вкладка **Comments** в сайдбаре (иконка облачка), так же открывается по клику
  на название трека (нижний бар / правая панель) и кнопкой 💬 на прогресс-баре.
- Обложка + название + автор, визуализатор с **всплывающим пузырём** текущего
  комментария (анимация входа/выхода), список всех комментариев с подсветкой
  проигранного и таймкодами. Клик по комментарию — перемотка на этот момент.
- Комментарии тянутся с SoundCloud API.

### 📥 Офлайн-кэш
- Кнопка скачивания (↓) на каждом SoundCloud-треке в списках — сохраняет MP3
  локально в `userData/offline`. Скачанный трек играет из кэша без интернета.
- Вкладка **Скачанное** в сайдбаре → «Моя музыка». В настройках → Storage:
  количество и размер кэша с кнопкой очистки.

### 🎮 Discord Rich Presence
- Показывает в профиле Discord: название трека, `by Автор`, статус играет/пауза,
  таймер воспроизведения. **Обложка трека** вместо иконки приложения.
- Настройка в Settings → Discord: тумблер + поле Application ID (создаётся на
  discord.com/developers). Без падений, если Discord не запущен.

### ⚡ Улучшения
- **Панель очереди**: drag-and-drop переупорядочивание, удаление отдельных треков,
  фильтр-строка над очередью.
- **История поиска** (последние запросы в поиске по строчке).
- SoundCloud-треки теперь проходят через общий Web Audio граф (EQ + живой
  визуализатор вместо синтетического), без глушения из-за CORS.

## [Unreleased]

### 📱 Мобильный таргет (iOS + Android, Capacitor)

- **Кроссплатформенный мобильный порт** на Capacitor: тот же web-бандл
  (`npm run build:mobile`) ставится и на iOS, и на Android. Платформо-зависимый
  слой (`mobile/src/api/*`) написан через `Capacitor.isNativePlatform()` +
  `CapacitorHttp` / `Filesystem` / `StatusBar` — один код на обе платформы.
- **Android**: `@capacitor/android`, секция `android` в `capacitor.config.ts`,
  нативные правки скриптом `mobile/scripts/patch-android.sh` (права на фоновое
  аудио и медиа-уведомление в манифесте + иконки из логотипа). Фоновое аудио и
  уведомление с кнопками prev/next обеспечивает системный WebView через Media
  Session API — без нативного кода транспорта (легче, чем iOS).
- **Сборка Android**: debug-`.apk` на Linux-раннере GitHub Actions
  (`.github/workflows/android.yml`) — без Mac, без секретов подписи, ставится
  напрямую на телефон, без ограничения 7 дней (как у iOS-сайдлоада).
- Документация: `mobile/android-notes.md`.

## [0.1.0] — 2026-06-20

Первый публичный билд. Ниже — полный обзор возможностей.

### 🎧 Воспроизведение и источники
- **Локальные файлы** — скан папок, чтение тегов и обложек (`music-metadata`),
  отдача через собственный протокол `media://`. Обложки кэшируются в `userData`.
- **SoundCloud** — поиск треков и артистов, стрим через публичный API:
  - progressive (MP3) и **HLS** (через `hls.js` + MSE);
  - `client_id` подбирается автоматически (скрейпится на лету, авто-рефреш на 401);
  - вход в аккаунт SoundCloud → персональные данные (см. ниже).
- **Единое ядро** — очередь, плеер-бар, лайки, плейлисты не зависят от источника.
- **Плеер-бар**: play/pause, prev/next, перемотка по волновой дорожке, громкость,
  shuffle, repeat (off/all/one), обложка и лайк текущего трека.
- **Очередь** персистится и восстанавливается после перезапуска (на паузе).

### 📚 Библиотека и навигация
- Страницы: **Home**, **Explore** (поиск SoundCloud + артисты), **Activity**
  (статистика и лента), **Artist**, **Mix**, **Info** (источники), **Profile**.
- **Your Likes** — лайки треков любого источника (локальные + SoundCloud).
- **Recently Played** — история прослушиваний.
- **Плейлисты** — создание/переименование/удаление, добавление треков из любого
  места (поповер «+»), обложки-миниатюры в сайдбаре.
- **Resizable** сайдбар и правая панель, минимальный размер окна 1280×800.

### 🎚️ Визуализатор (синхрон с музыкой)
- Единый Web Audio граф с `AnalyserNode`; полоски анимируются императивно через
  `requestAnimationFrame` + `transform: scaleY()` (без ре-рендеров React).
- Три инстанса с разной «силой реакции»: нижний прогресс-бар (едва), правая
  колонка (слабо), полноэкранный (заметно).
- Локальные файлы — **реальный FFT**; SoundCloud (cross-origin CDN) — мягкое
  синтетическое «дыхание» (звук там не глушится Web Audio).

### 🎤 Тексты песен и караоке
- Источник — **LRCLIB** (бесплатно, без ключа): синхронизированный LRC →
  построчная караоке-подсветка по позиции + автоскролл + клик по строке = seek.
- Фолбэк на **Genius** (plain-текст) с оценочной авто-разбивкой по времени.
- **Ручная синхронизация** (SyncEditor) — свои тайминги, защищённые от перезаписи.
- Кэш текстов в `userData/lyrics`.

### 🖥️ Полноэкранный плеер
- Открывается из плеер-бара; обложка, караоке-текст, волновая дорожка, транспорт.
- Фон — градиент из палитры обложки **или** ваша картинка (см. оформление).
- Под текстом — растушёванная **стеклянная подложка** (`backdrop-filter`) для
  читаемости поверх любого фона; панель плавно заходит под обложку.
- Кинематографичное появление фона без мигания (предзагрузка + decode-гейт).

### 👤 Личный кабинет
- Чип профиля в шапке сайдбара (аватар + ник) → страница профиля.
- Редактируемые **имя** и **аватар** (с кадрированием по кругу); локальные
  значения перекрывают данные из SoundCloud, есть сброс «как в SoundCloud».
- Подключение/отключение **SoundCloud** прямо из профиля.
- Статистика: треков сыграно, время прослушивания, лайки, плейлисты.
- Блоки **Недавние артисты** и **Недавнее** (карточки треков).
- «Понтовые» счётчики (подписчики / прослушивания / ★ оценка) — редактируются
  кликом, для красоты.

### 🎨 Оформление
- **Темы**: Green, Purple, Blue, Pink + **кастомный акцент** — собственный
  HSV-пикер в стиле приложения (квадрат, hue-слайдер, hex, пресеты); производные
  оттенки выводятся через `color-mix`. Перетаскивание не лагает (коммит на отпускании).
- **Кастомный фон**: выбор картинки + **недеструктивное кадрирование**
  (пан/зум, live-превью), **область применения** — полноэкранный / интерфейс /
  везде. Полупрозрачные панели поверх фона.
- **Компактный режим**, размер текста караоке (S/M/L).

### 🌍 Локализация
- Полностью на **двух языках**: English и Русский (переключается в настройках).

### ⚙️ Настройки и хранение
- Настройки: язык, тема/цвет, компактный режим, фон + область + кадрирование,
  размер караоке, восстановление сессии, фолбэк на Genius, автозапуск, очистка
  кэшей (тексты / миксы).
- Персист в `localStorage` (очередь, громкость, история, тема, профиль, фон…) и в
  `userData` (лайки, плейлисты, кэш обложек/текстов, токен SoundCloud).

### 📦 Платформа и сборка
- Сборка для Windows: установщик **NSIS** + **portable** (`npm run dist:win`).
- Системная иконка — неоновый логотип «L» в аккуратной squircle-рамке.

### 🔒 Приватность
- В коде нет захардкоженных секретов; `client_id` берётся на лету, OAuth-токен
  SoundCloud перехватывается из вашего же логина и хранится **локально**
  (`%APPDATA%/latency-player/soundcloud.json`) — в репозиторий не попадает.

### 🧱 Границы по легальности
- Локальные файлы — без ограничений; SoundCloud — публичный API; Spotify/YouTube —
  только официальные SDK/embed. Обход DRM и риппинг защищённого потока не делается.
